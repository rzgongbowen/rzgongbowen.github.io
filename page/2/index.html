java面试

2020-07-24

  • Java基础

    1. Java的跨平台原理

      • Java通过不同的操作系统、不同的版本、不同位数的java虚拟机（jvm）来屏蔽不同的系统指令集差异而对外提供统一的接口（java API），对于我们普通的java 开发者而言,只需要按照接口开发即可。如果系统需要部署到不同的环境时，只需在系统上安装对应的虚拟机即可。

    2. Java开发环境需要些什么

      • 适用开发环境的jdk
        • 配置JAVA_HOME
      • 对应的开发环境
      • web服务器（tomcat）

    3. Java中int占几个字节

      • Int占4个字节，32位
      • Boolean 1位

    4. Java面向对象的特征

      • 四大基本特征：封装、抽象、继承、多态
        • 封装性，即将对象封装成一个高度自治和相对集中的封闭的个体。
        • 抽象性，就是找出事物的相似和共性之处，然后将这些事物归为一个类，这个类只考虑事物的相似和共性之处，就是把现实生活中的对象，抽象为类。
        • 继承性，把这个已经存在的类所定义的内容作为自己的内容，并可以加入若干新的内容，或修改原来的方法使之变为更合适特殊的需要，遗产继承。
        • 多态性，指程序中定义的引用变量所指向的具体类型和通过该引用变量发出的方法调用编程时不确定，而是到程序运行过程中才确定，即一个引用变量到底指向哪一个的实例对象，该引用变量发出的方法调用到底是哪个类中实现的方法，必须由程序运行期间才决定。

    5. 装箱和拆箱

      • 基本数据类型：8种基本数据类型，boolean、int、float等
      • 包装类型：每个基本数据类型都一一对应一个包装类型
        • boolean –> Boolean
        • int –> Integer
      • 装箱：把基本的数据类型转换为对应的包装类型
        • Integer.valueOf(1);//手动装箱
        • Integer i = 1;//自动装箱
      • 拆箱：就是把包装类型转换为基本数据类型
        • Integer i = 1；int j = i;//自动拆箱 int j = i.intValue();//手动拆箱
      • 为什么？
        • Java是面向对象的语言，而基本数据类型不具备面向对象的特性

    6. “==”和 equals 方法的区别

      • “==”用来判断两个变量之间的值是否相等，变量就可以分为基本数据类型变量，引用类型。
        • 基本数据类型：直接比较值
        • 引用类型：比较对应的引用的内存的首地址
      • equals 用来比较两个对象是否长得一样，判断两个对象的某些特征是否一样。

    7. String和StringBuilder StringBuffer的区别

      • 在java中提供了三个类String StringBuilder StringBuffer来表示和操作字符串，字符串就是多个字符的集合；
        • String是内容不可变的字符串。String底层使用了一个不可变的字符数组（final char[]）
        • StringBuilder StringBuffer，是内容可以改变的字符串。StringBuilder StringBuffer底层使用的是可变的字符数组（没有使用final修饰）
      • StringBuilder是线程不安全的，效率较高；StringBuffer是线程安全的，效率较低。

    8. Java中的集合

      • Java集合中的集合分为value，key-value(collection map)两种
        • 存储值分为List和Set
          • List是有序的，可以重复。
          • set是无序的，不可以重复。根据equals和hashcode判断，也就是说一个对象要存储在set中，必须重写equals和hashCode方法。
        • 存储key-value的为map

    9. ArrayList和LinkedList的区别

      • ArrayList底层是数组，LinkedList底层是链表
        • ArrayList查询快，增删慢
          • 数组是内存中一块连续的内存，插入或者删除数据需要移动内存
        • LinkedList查询慢，增删快
          • 插入或者删除时，不需要移动内存，只需改变引用指向即可
      • ArrayList适用查询多，增删少的情况；LinkedList适用于查询少，增删多的情况

    10. HashMap和HashTable区别

      • 相同点：
        • HashMap和HashTable都可以用来存储key-value的数据
      • 区别：
        • HashMap可以把null作为key或者value的，而HashTable不可以
        • HashMap是线程不安全的，效率高；HashTable是线程安全的，效率低
      • 既想线程安全又想效率高？
        • 适用ConcurrentHashMap，把整个Map分为N个Segment（类似HashTable）

    11. 实现一个拷贝文件的工具类适用字节流还是字符流？

      • 为考虑通用性，要使用字节流

    12. 线程的几种实现方式

      • 通过继承Thread类实现一个线程
        • 继承扩展性不高，java只支持单继承，如果一个继承了Thread就不能再继承其他的类了
      • 通过实现Runnable接口实现一个线程
      • 怎么启动？
        • 启动线程使用start方法，启动之后执行的是run方法
      • 区分线程
        • thread.set(设置一个线程名称);

    13. 什么是设计模式？常用的设计模式有哪些？

      • 设计模式是经过前人无数次的实践总结得出的，设计模式可以反复使用、可以解决特定问题的设计方法
      • 单例（饱汉模式、饥汉模式）：
        • 构造方法私有化
        • 在自己的类中创建一个单实例（饱汉模式是一出来就创建单实例，饥汉模式是需要的时候再创建）
        • 提供一个方法获取该实例对象（创建时需要进行方法同步）
      • 工厂模式：对象的创建交给一个工厂去创建
      • 代理模式
      • 包装模式

    14. 线程池的作用

      1. 限定线程的个数，不会由于线程过多而导致系统运行缓慢或崩溃
      2. 线程池不需要每次都去创建或销毁，节约资源
      3. 线程池不需要每次都去创建，响应时间更快

  • Javaweb部分

    1. get和post请求区别

      • get,post,put,delete对应着资源的查，改，增，删。
      • get一般用于获取/查询数据；post一般用于更新资源信息
      • get请求提交的数据会在地址栏显示出来，而post请求的数据则不会在地址栏显示
      • post请求的安全性要比get的安全性高

    2. Servlet是什么

      • 全称是Java Servlet，是用Java编写的服务器端程序。而这些Servlet都要实现Servlet这个接口。主要功能在于交互式地浏览和修改数据，生成动态Web内容。

    3. Servlet的生命周期

      • Servlet有良好的生存期的定义
      • 加载Servlet的class->实例化Servlet->调用Servlet的init完成初始化->响应请求(Servlet的service方法)->Servlet容器关闭时(Sevlet的destory方法)

    4. Servlet API中forward()与redirect()的区别

      • forward是服务器端的转向而redirect是客户端的跳转
      • 使用forward浏览器的地址不会发生改变，而redirect会发生改变
      • forward是一次请求中完成，而redirect是重新发起请求
      • forward是服务器端完成，而不用客户端重新发起请求，效率较高

    5. JSP和Servlet有哪些相同点和不同点，他们之间的联系是什么

      • JSP是Servlet技术的扩展，所有的jsp文件都会被翻译一个继承HTTPServlet的类，也就是jsp最终也是一个Servlet，这个Servlet对外提供服务
      • Servlet如果要实现html的功能，必须使用Writer输出对应的html，比较麻烦。而jsp的情况是java和html可以组合一个扩展名为.jsp的文件，做界面展示比较方便而嵌入逻辑比较复杂
      • Servlet和JSP最主要的不同点在于JSP侧重于视图，Servlet主要用于控制逻辑

    6. jsp有哪些内置对象，作用分别是什么

      • 9个内置对象
        1. request 用户端请求。此请求会包含来自GET/POST请求的参数
        2. response 网页传回用户端的回应
        3. pageContext 网页的属性是在这里管理
        4. session 与请求有关的会话期
        5. application Servlet 正在执行的内容
        6. out 用来传递回应的输出
        7. config Servlet 的构架部件
        8. page JSP 网页本身
        9. exception 针对错误网页，未捕捉的例外
      • 四大作用域：pageContext request session application 可以通过jstl从四大作用域取值

    7. session和cookie的区别

      • session和cookie都是会话跟踪技术，cookie通过在客户端记录信息确定用户身份，session通过在服务器端记录信息确定用户信息。但是session的实现依赖cookie，sessionId（session的唯一标识需要存放在客户端）
      • cookie和session的区别
        1. cookie数据存放在客户的浏览器，session数据存放在服务器上
        2. cookie不是很安全，别人可以分析存放在本地的cookie并进行cookie欺骗，考虑到安全应当使用session
        3. session会在一定时间保存在服务器，当访问增多，会比较占用你服务器的性能，考虑到减轻服务器的性能，应当使用cookie
        4. 单个cookie保存的数据不能超过4k，很多浏览器都限制一个站点最多保存20个cookie
        5. 将登陆信息存放在session，其他信息如果需要保留，可以放在cookie中，比如购物车

    8. MVC的各个部分都有哪些技术来实现

      • M(Model) 模型 javabean
      • V(View) 视图 HTML jsp
      • C(Control) 控制器 Servlet

  • 数据库部分

    1. 数据库分类

      • 关系型数据库
        • mysql、oracle、sqlserver等
      • 非关系型数据
        • redis、hadoop等

    2. 关系型数据库的三范式

      • 范式就是规范，要想满足第二范式必须满足第一范式，要想满足第三范式必须满足第二范式
        • 第一范式是指数据库中的每一列都是不可分割的基本数据项，同一个列中不能有多个值
        • 第二范式要求数据表中的每一个行必须可以被唯一地区分（主键）
        • 第三范式要求一个数据库表中不包含已在其他表中已包含的非主关键字信息（外键）

    3. 事务四大基本特征

      • 事务是并发控制的单位，用用户定义的一个操作序列。这些操作要么都做，要么都不做，是一个不可分割的工作单位
        • 转账，A->B,A扣钱成功，B加钱成功，才真正转账成功
      • 事务必须满足四大特征
        • 原子性：事务内操作不可分割，要么都成功，要么都失败
        • 一致性：要么都成功，要么都失败，后面失败的要对前面成功的操作进行回滚
        • 隔离性：一个事务开始后，不能让其他事务干扰
        • 持久性/持续性：表示事务开始了，就不能终止

    4. mysql数据库的默认的最大连接数

      • 默认100
      • 可以自行设置

    5. mysql的分页

      • 为什么需要分页？
        • 数据多，不可能完全显示数据，进行分段显示
      • mysql使用关键字limit来进行分页 limit offset,size 表示从多少索引去多少位

    6. 触发器的使用场景

      • 触发器，需要有触发条件，当条件满足以后做什么操作

    7. 数据库存储过程的使用场景

      1. 存储过程只在创建时进行编译，大大提高数据库执行速度
      2. 复杂的业务逻辑需要多条sql语句，如果将这些操作放在一个存储过程中，那么客户机和服务器之间的网络传输就是大大减少，降低网络负载
      3. 存储过程创建一次便可以重复使用，从而大大减少数据库开发人员的工作量
      4. 安全性高，存储过程可以屏蔽对底层数据库对象的直接访问

    8. 用jdbc怎么调用存储过程

      1. 加载驱动
      2. 获取连接
      3. 设置参数
      4. 执行
      5. 释放连接

    9. 对jdbc的理解

      • Java只定义接口，让数据库厂商自己实现接口，对于我们开发者而言，只需要导入对应厂商开发的实现即可。然后以接口方式进行调用（mysql+mysql驱动（实现）+jdbc）

    10. JDBC中的PreparedStatement相比Statement的好处

      • 大多数我们都使用PreparedStatement
        • PreparedStatement是预编译的，比Statement速度快
        • 代码的可读性和可维护性
        • PreParedStatement可以防止SQL注入攻击，而Statement却不能

    11. 数据库连接池的作用

      1. 限定数据库的个数，不会由于数据库连接过多而导致系统运行缓慢或崩溃
      2. 数据库连接不需要每次都去创建或销毁，节约资源
      3. 数据库连接不需要每次都去创建，响应时间更快

      1. 限定线程的个数，不会由于线程过多而导致系统运行缓慢或崩溃
      2. 线程池不需要每次都去创建或销毁，节约资源
      3. 线程池不需要每次都去创建，响应时间更快

  • Javaweb部分

    1. get和post请求区别

      • get,post,put,delete对应着资源的查，改，增，删。
      • get一般用于获取/查询数据；post一般用于更新资源信息
      • get请求提交的数据会在地址栏显示出来，而post请求的数据则不会在地址栏显示
      • post请求的安全性要比get的安全性高

    2. Servlet是什么

      • 全称是Java Servlet，是用Java编写的服务器端程序。而这些Servlet都要实现Servlet这个接口。主要功能在于交互式地浏览和修改数据，生成动态Web内容。

    3. Servlet的生命周期

      • Servlet有良好的生存期的定义
      • 加载Servlet的class->实例化Servlet->调用Servlet的init完成初始化->响应请求(Servlet的service方法)->Servlet容器关闭时(Sevlet的destory方法)

    4. Servlet API中forward()与redirect()的区别

      • forward是服务器端的转向而redirect是客户端的跳转
      • 使用forward浏览器的地址不会发生改变，而redirect会发生改变
      • forward是一次请求中完成，而redirect是重新发起请求
      • forward是服务器端完成，而不用客户端重新发起请求，效率较高

    5. JSP和Servlet有哪些相同点和不同点，他们之间的联系是什么

      • JSP是Servlet技术的扩展，所有的jsp文件都会被翻译一个继承HTTPServlet的类，也就是jsp最终也是一个Servlet，这个Servlet对外提供服务
      • Servlet如果要实现html的功能，必须使用Writer输出对应的html，比较麻烦。而jsp的情况是java和html可以组合一个扩展名为.jsp的文件，做界面展示比较方便而嵌入逻辑比较复杂
      • Servlet和JSP最主要的不同点在于JSP侧重于视图，Servlet主要用于控制逻辑

    6. jsp有哪些内置对象，作用分别是什么

      • 9个内置对象
        1. request 用户端请求。此请求会包含来自GET/POST请求的参数
        2. response 网页传回用户端的回应
        3. pageContext 网页的属性是在这里管理
        4. session 与请求有关的会话期
        5. application Servlet 正在执行的内容
        6. out 用来传递回应的输出
        7. config Servlet 的构架部件
        8. page JSP 网页本身
        9. exception 针对错误网页，未捕捉的例外
      • 四大作用域：pageContext request session application 可以通过jstl从四大作用域取值

    7. session和cookie的区别

      • session和cookie都是会话跟踪技术，cookie通过在客户端记录信息确定用户身份，session通过在服务器端记录信息确定用户信息。但是session的实现依赖cookie，sessionId（session的唯一标识需要存放在客户端）
      • cookie和session的区别
        1. cookie数据存放在客户的浏览器，session数据存放在服务器上
        2. cookie不是很安全，别人可以分析存放在本地的cookie并进行cookie欺骗，考虑到安全应当使用session
        3. session会在一定时间保存在服务器，当访问增多，会比较占用你服务器的性能，考虑到减轻服务器的性能，应当使用cookie
        4. 单个cookie保存的数据不能超过4k，很多浏览器都限制一个站点最多保存20个cookie
        5. 将登陆信息存放在session，其他信息如果需要保留，可以放在cookie中，比如购物车

    8. MVC的各个部分都有哪些技术来实现

      • M(Model) 模型 javabean
      • V(View) 视图 HTML jsp
      • C(Control) 控制器 Servlet

  • 数据库部分

    1. 数据库分类

      • 关系型数据库
        • mysql、oracle、sqlserver等
      • 非关系型数据
        • redis、hadoop等

    2. 关系型数据库的三范式

      • 范式就是规范，要想满足第二范式必须满足第一范式，要想满足第三范式必须满足第二范式
        • 第一范式是指数据库中的每一列都是不可分割的基本数据项，同一个列中不能有多个值
        • 第二范式要求数据表中的每一个行必须可以被唯一地区分（主键）
        • 第三范式要求一个数据库表中不包含已在其他表中已包含的非主关键字信息（外键）

    3. 事务四大基本特征

      • 事务是并发控制的单位，用用户定义的一个操作序列。这些操作要么都做，要么都不做，是一个不可分割的工作单位
        • 转账，A->B,A扣钱成功，B加钱成功，才真正转账成功
      • 事务必须满足四大特征
        • 原子性：事务内操作不可分割，要么都成功，要么都失败
        • 一致性：要么都成功，要么都失败，后面失败的要对前面成功的操作进行回滚
        • 隔离性：一个事务开始后，不能让其他事务干扰
        • 持久性/持续性：表示事务开始了，就不能终止

    4. mysql数据库的默认的最大连接数

      • 默认100
      • 可以自行设置

    5. mysql的分页

      • 为什么需要分页？
        • 数据多，不可能完全显示数据，进行分段显示
      • mysql使用关键字limit来进行分页 limit offset,size 表示从多少索引去多少位

    6. 触发器的使用场景

      • 触发器，需要有触发条件，当条件满足以后做什么操作

    7. 数据库存储过程的使用场景

      1. 存储过程只在创建时进行编译，大大提高数据库执行速度
      2. 复杂的业务逻辑需要多条sql语句，如果将这些操作放在一个存储过程中，那么客户机和服务器之间的网络传输就是大大减少，降低网络负载
      3. 存储过程创建一次便可以重复使用，从而大大减少数据库开发人员的工作量
      4. 安全性高，存储过程可以屏蔽对底层数据库对象的直接访问

    8. 用jdbc怎么调用存储过程

      1. 加载驱动
      2. 获取连接
      3. 设置参数
      4. 执行
      5. 释放连接

    9. 对jdbc的理解

      • Java只定义接口，让数据库厂商自己实现接口，对于我们开发者而言，只需要导入对应厂商开发的实现即可。然后以接口方式进行调用（mysql+mysql驱动（实现）+jdbc）

    10. JDBC中的PreparedStatement相比Statement的好处

      • 大多数我们都使用PreparedStatement
        • PreparedStatement是预编译的，比Statement速度快
        • 代码的可读性和可维护性
        • PreParedStatement可以防止SQL注入攻击，而Statement却不能

    11. 数据库连接池的作用

      1. 限定数据库的个数，不会由于数据库连接过多过多而导致系统运行缓慢或崩溃
      2. 数据库连接不需要每次都去创建或销毁，节约资源
      3. 数据库连接不需要每次都去创建，响应时间更快
